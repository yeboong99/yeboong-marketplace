# 커밋 메시지 작성 팁 및 주의사항

---

## ✅ 좋은 커밋 메시지

```
feat: 사용자 프로필 이미지 업로드 기능 추가
```
→ 타입 명확, 제목 간결, 50자 이내

```
fix: 장바구니에서 수량 0일 때 삭제 버튼 미표시 버그 해결

수량이 0이 되면 조건부 렌더링이 false를 반환하여
버튼이 사라지던 문제. 조건식을 quantity !== null로 수정

Fixes #204
```
→ 본문에 원인과 해결 방법 모두 포함, 이슈 참조

```
feat!: 인증 API 응답 구조 변경
```
→ Breaking Change를 `!`로 명시하여 Major 버전 트리거

---

## ❌ 나쁜 커밋 메시지

```
fix: 버그 수정
```
→ 어떤 버그인지 전혀 알 수 없음

```
feat: 사용자가 프로필 페이지에서 자신의 이미지를 업로드할 수 있고 크롭할 수 있으며 저장할 수 있는 기능을 추가함
```
→ 제목이 너무 길고, "추가함"은 명령형이 아님

```
feat: 로그인 기능 추가 및 장바구니 버그 수정 및 문서 업데이트
```
→ 여러 변경사항이 혼합됨 → 커밋을 분리해야 함

```
feat: 결제 API v2 적용
```
→ 실제로는 Breaking Change인데 `!` 없이 작성 → Major 버전 누락

---

## 주의사항

- **하나의 커밋 = 하나의 논리적 변경사항**: 이것이 지켜지지 않으면 시맨틱 버저닝이 올바르게 동작하지 않고, 코드 리뷰와 롤백도 어려워집니다.
- **커밋 분리가 필요하면 반드시 먼저 제안**: 사용자에게 확인 없이 단일 커밋으로 작성하지 마세요.
- **Breaking Change 누락 금지**: `!` 또는 `BREAKING CHANGE:` 없이 작성하면 자동 태깅에서 Major 버전이 올라가지 않습니다.
- **제목은 명령형**: "추가했음" ❌ → "추가" ✅ / "수정함" ❌ → "수정" ✅
- **마침표 없음**: 제목 끝에 마침표를 붙이지 않습니다.
- **한국어로 작성**: 영어로 작성하지 않습니다.

## 추가 예시

### 커밋 분리가 필요한 경우

**상황**: 로그인 기능 추가 + 장바구니 버그 수정이 동시에 staged

**올바른 응답**:
```
⚠️ 변경사항에 여러 논리적 단위가 포함되어 있습니다. 아래와 같이 커밋을 분리하는 것을 강력히 권장합니다:

커밋 1: feat: 사용자 로그인 폼 컴포넌트 추가
  - 관련 파일: src/components/LoginForm.tsx, src/hooks/useAuth.ts

커밋 2: fix: 장바구니 수량 0일 때 삭제 버튼 미표시 버그 해결
  - 관련 파일: src/components/CartItem.tsx

분리하여 커밋하시겠습니까?
```

### refactor (버전 변화는 default_bump 따름)

```
refactor: 사용자 인증 로직 모듈화

- 중복되던 토큰 검증 코드를 별도 함수로 분리
- 인증 관련 유틸리티 함수 통합
```

### chore (의존성 업데이트)

```
chore: Next.js 14.0.0으로 업데이트
```

### docs (문서 수정)

```
docs: README에 로컬 개발 환경 설정 가이드 추가
```

### test (테스트 추가)

```
test: 결제 서비스 단위 테스트 추가

- 카드 결제 성공/실패 케이스 커버
- 환불 로직 엣지 케이스 추가
```

### Breaking Change — 푸터 방식 (상세 설명 필요 시)

```
feat: 결제 API v2 도입

새로운 결제 플로우 및 웹훅 지원 추가

BREAKING CHANGE: /api/v1/payment 엔드포인트가 제거되었습니다.
반드시 /api/v2/payment로 마이그레이션해야 합니다.
```
